# Figma wireframes

The low-fidelity wireframes for both apps live in this Figma file:

https://www.figma.com/design/1qRZGRDZYtLSEo26wMv1XB/Case-Study

The file is organised into two pages:

## Page 1 — PurchaseRequest App

| Frame | Purpose |
| --- | --- |
| `Purchase Requests` | Main list, employee view |
| `PR · 01 — Login` | Login screen |
| `PR · 02 — Purchase Requests (Finance view)` | Main list when signed in as finance — adds an Actions column with Approve buttons |
| `PR · 03 — Create New Purchase Request` | "Create" modal overlay |
| `PR · 04a — View Purchase Request (Employee)` | View modal for the request author; can edit while `initiated`, then send for approval |
| `PR · 04b — View Purchase Request (Finance review)` | View modal for finance reviewers; shows Approve / Reject |

## Page 2 — Invoice App

| Frame | Purpose |
| --- | --- |
| `INV · 01 — Login (Finance only)` | Login screen with a note that only `finance` users are allowed |
| `INV · 02 — Invoices list` | Main list with summary stat cards (totals by status) and the invoice table |
| `INV · 03 — Upload Invoice` | Upload modal with PDF drop zone and form fields |
| `INV · 04 — Edit Invoice` | Edit modal with attachment preview, form fields, status segments, and a "linked purchase request" callout |

The wireframes are deliberately low-fidelity. Treat them as a starting point — feel free to push back on the layout if you have better ideas.
